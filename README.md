### **DESCRIPTION OF THE TASK**

As part of my internship with **CODTECH IT SOLUTIONS** under the **Full Stack Web Development** domain, I completed **Task 1: API Integration**, where I developed a **Weather Forecast Web Application**. The goal of this task was to create a responsive and interactive webpage that fetches and displays real-time weather information using a **public Weather API**.  

This project demonstrates the concept of **API integration**, which allows web applications to interact with external data sources. The Weather App provides users with the ability to enter a city name and instantly view weather information such as **temperature, humidity, wind speed, and general conditions**. The data is fetched from the **OpenWeatherMap API**, processed using **JavaScript**, and displayed neatly on the user interface.  

The main purpose of this task was to understand how to make HTTP requests, handle JSON responses, manage asynchronous operations, and dynamically update web content using JavaScript. This project enhanced my practical understanding of API handling and front-end logic building.

### **WORKFLOW / IMPLEMENTATION**

1. **Project Setup:**  
   The project starts with setting up an HTML page containing a search box and a ‚ÄúGet Weather‚Äù button. The user can input a city name to retrieve its current weather information.  

2. **API Integration:**  
   I registered on **OpenWeatherMap** and obtained a free API key. This key is used to make API calls to fetch weather details based on the city name provided by the user.  

   Example API endpoint:
   https://api.openweathermap.org/data/2.5/weather?q={city name}&appid={API key}&units=metric

    
3. **Fetching Data:**  
Using JavaScript‚Äôs `fetch()` method, a request is sent to the API endpoint. The response is received in JSON format and parsed to extract weather-related data such as temperature, humidity, wind speed, and condition description.  

4. **Dynamic Display:**  
The data is displayed dynamically on the webpage using **DOM manipulation**. When the user searches for another city, the data updates automatically without reloading the page.  

5. **Error Handling:**  
If an invalid city is entered or there is an issue with fetching data, the app displays a friendly error message to inform the user.  

6. **Styling and Responsiveness:**  
The application is styled using CSS to create a clean, modern look with gradient backgrounds and smooth transitions. It is also responsive, adapting well to different screen sizes like mobile, tablet, and desktop.  

7. **Testing and Validation:**  
The app was tested using multiple city names to ensure the accuracy of results and smooth user interaction.
